# Domain Model Specification (Stage 2) - Part 10: Appendix

## 10. APPENDIX

### Glossary of Domain Terms (อภิธานศัพท์คำศัพท์โดเมน)

- **Branching Narrative**: โครงสร้างเรื่องที่การเลือกของผู้เล่นสร้างเส้นทางเรื่องที่แตกต่างกัน
- **Credit**: สกุลเงินในเกมที่ใช้สำหรับเข้าถึงตัวเลือกเรื่องและเนื้อหาพรีเมียม
- **DAG Editor**: เครื่องมือแก้ไขภาพกราฟิก Directed Acyclic Graph สำหรับโครงสร้างเรื่อง
- **Ending Type**: การจำแนกจุดสิ้นสุดเรื่อง (normal, special, perfect)
- **Guest User**: ผู้เล่นชั่วคราวโดยไม่มีบัญชีถาวรหรือการบันทึกความคืบหน้า
- **Interactive Fiction**: รูปแบบเรื่องที่ผู้อ่านตัดสินใจส่งผลต่อผลลัพธ์
- **Multimedia Segment**: องค์ประกอบเนื้อหาตามเวลาที่รวมวิดีโอ รูปภาพ และข้อความ
- **Path Tracking**: การบันทึกการเดินทางของผู้ใช้ผ่านตัวเลือกและฉากเรื่อง
- **Prerequisites**: เงื่อนไขที่ต้องเป็นไปเพื่อเข้าถึงเนื้อหาหรือตัวเลือกบางอย่าง
- **Refill Interval**: ช่วงเวลาระหว่างการเพิ่มเครดิตอัตโนมัติในกระเป๋าเงินผู้ใช้
- **Scene**: ส่วนของเรื่องแต่ละส่วนที่มีเนื้อหาและตัวเลือกที่มี
- **Story Graph**: การแสดงภาพโครงสร้างเรื่องด้วยโหนดและการเชื่อมต่อ
- **Unlock Condition**: ข้อกำหนดเฉพาะสำหรับเข้าถึงความสำเร็จ อวาตาร์ หรือเนื้อหา

### Notes for Future API Design (โน้ตสำหรับการออกแบบ API ในอนาคต)

#### Authentication and Authorization (การตรวจสอบสิทธิ์และการอนุญาต)
- การจัดการเซสชันต้องรองรับทั้งผู้ใช้ที่ลงทะเบียนและผู้มาเยี่ยม
- การควบคุมการเข้าถึงตามบทบาท (RBAC) จำเป็นสำหรับฟังก์ชันผู้ดูแลระบบ
- โทเค็น JWT พร้อมเวลาหมดอายุที่เหมาะสม
- การจำกัดอัตราตามประเภทและกิจกรรมของผู้ใช้

#### Data Consistency Patterns (รูปแบบความสอดคล้องข้อมูล)
- ใช้การควบคุมการพร้อมกันแบบมุ่งมั่นดีสำหรับการแก้ไขเรื่อง
- ใช้ธุรกรรมฐานข้อมูลสำหรับการเคลื่อนไหวเครดิต
- การจัดหาแหล่งที่มาของเหตุการณ์สำหรับร่องรอยการตรวจสอบและการวิเคราะห์
- กลยุทธ์แคชสำหรับข้อมูลเรื่องที่เข้าถึงบ่อย

#### Performance Considerations (การพิจารณาประสิทธิภาพ)
- ใช้การแบ่งหน้าสำหรับชุดข้อมูลขนาดใหญ่ (เรื่อง, รีวิว, การวิเคราะห์)
- การผสานรวม CDN สำหรับการส่งมอบสื่อ
- การสร้างดัชนีฐานข้อมูลสำหรับการค้นหาและกรองเรื่อง
- การประมวลผลแบบอะซิงโครนัสสำหรับการสร้างเนื้อหา AI

#### Scalability Requirements (ข้อกำหนดความสามารถในการขยายตัว)
- รองรับเซสชันเรื่องพร้อมกัน (ประมาณ 10,000+ พร้อมกัน)
- การขยายตัวแนวนอนสำหรับการส่งมอบสื่อเรื่อง
- แยกเรพลิกาอ่านสำหรับคิวรีการวิเคราะห์
- การประมวลผลตามคิวสำหรับงานพื้นหลัง

### Special Implementation Requirements (ข้อกำหนดการนำไปใช้งานพิเศษ)

#### Real-time Features (ฟีเจอร์เรียลไทม์)
- การเชื่อมต่อ WebSocket สำหรับการทำงานร่วมกันของเรื่องแบบสด
- การแจ้งเตือนแบบพุชสำหรับการเติมเต็มเครดิตและความสำเร็จ
- แดชบอร์ดการวิเคราะห์เรียลไทม์พร้อมอัปเดตสด
- การส่งข้อความทันทีสำหรับการทำงานร่วมกันของผู้ดูแลระบบ

#### Security Considerations (การพิจารณาความปลอดภัย)
- การทำความสะอาดข้อมูลนำเข้าสำหรับเนื้อหาที่ผู้ใช้สร้างทั้งหมด
- การจำกัดอัตราบนธุรกรรมเครดิตเพื่อป้องกันการใช้ในทางที่ผิด
- การสแกนเนื้อหาสำหรับวัสดุที่ไม่เหมาะสม
- การบันทึกร่องรอยการตรวจสอบสำหรับการกระทำผู้ดูแลระบบทั้งหมด

#### Integration Points (จุดผสานรวม)
- เกตเวย์การชำระเงินสำหรับการซื้อเครดิต
- บริการอีเมลสำหรับการแจ้งเตือนผู้ใช้
- API สื่อสังคมสำหรับฟีเจอร์การแชร์
- ผู้ให้บริการ CDN สำหรับการจัดเก็บและส่งมอบสื่อ
- บริการวิเคราะห์สำหรับการติดตามพฤติกรรมผู้ใช้

### Data Migration Strategy (กลยุทธ์การโยกย้ายข้อมูล)

#### Phase 1: Core Domain (โดเมนหลัก)
- ผู้ใช้และระบบการตรวจสอบสิทธิ์
- โครงสร้างเรื่องพื้นฐาน (ฉาก, ตัวเลือก)
- ระบบกระเป๋าเงินเครดิตและธุรกรรม

#### Phase 2: Enhanced Features (ฟีเจอร์ที่พัฒนาขึ้น)
- การนำไปใช้งานระบบความสำเร็จ
- ฟังก์ชันการจัดอันดับและรีวิว
- การผสานรวมการจัดการสื่อ

#### Phase 3: Advanced Features (ฟีเจอร์ขั้นสูง)
- เครื่องมือสร้างเนื้อหา AI
- การวิเคราะห์และรายงานขั้นสูง
- ฟีเจอร์สังคมและการแชร์

#### Phase 4: Optimization (การปรับแต่ง)
- การปรับแต่งประสิทธิภาพตามรูปแบบการใช้งาน
- ประเภทเนื้อหาและรูปแบบสื่อเพิ่มเติม
- การขยายเครื่องมือผู้ดูแลระบบและการทำงานอัตโนมัติ
